# 买家标签数据分析模型设计

## 📊 数据源

### 1. 订单表：`dunhill_t01_trade_line`

**关键字段：**

- `订单号`, `子订单号` - 订单唯一标识
- `买家昵称` - 关联字段
- `client_monthly_tag` - 新老客类型
- `最后付款时间` - 订单时间
- `成交总金额(gmv)` - 订单金额
- `退款金额(refund amount)` - 退款金额
- `退款类型` - 仅退款/退货退款
- `城市` - 地理位置
- `product_name` - 商品名称
- `天猫外部编码(sku)` - 商品SKU
- `category` - 商品品类
- `FP_MD` - FP=正价品, MD=折扣品

**计算公式：**

- 实际消费金额 = `gmv` - `refund_amount`
- 但只要有成交记录（即使退款）也算活跃

### 2. 聊天记录表：`chat_history`

**字段：**

- `user_nick` - 买家昵称
- `sender_nick` - 卖家昵称
- `msg_time` - 消息时间
- `msg_type` - 消息类型
- `content` - 消息内容

## 🏷️ 标签体系设计

### 一、基于订单数据的标签

#### 1.1 客户价值标签

**VIP等级** (与线下CRM保持一致 - Rolling 24个月)

```
计算逻辑：
- 计算 rolling 24个月(最近24个月) netsales = SUM(成交总金额 - 退款金额)
- 时间窗口：从当前日期向前推24个月
- 仅统计该窗口内的订单

等级划分：
- V3 (最高级): rolling_netsales >= 450,000
- V2: 150,000 <= rolling_netsales < 450,000
- V1: 50,000 <= rolling_netsales < 150,000
- V0: 30,000 <= rolling_netsales < 50,000
- Non-VIP: rolling_netsales < 30,000

示例SQL：
SELECT
    买家昵称,
    SUM(成交总金额 - 退款金额) as rolling_netsales
FROM dunhill_t01_trade_line
WHERE 最后付款时间 >= DATE_SUB(NOW(), INTERVAL 24 MONTH)
GROUP BY 买家昵称
```

**高价值客户** (High Value)

```
条件：
- L6M(近6个月)消费 >= 5,000
- OR 订单数 >= 3
- OR 单笔订单 >= 3,000
```

#### 1.2 折扣敏感度标签

**折扣敏感度** (分为3级)

```
计算逻辑：
- 正价品订单数 = COUNT(FP_MD = 'FP')
- 折扣品订单数 = COUNT(FP_MD = 'MD')
- 折扣品金额占比 = 折扣品总金额 / 总消费金额
- 折扣品数量占比 = 折扣品订单数 / 总订单数

分级标准：
- 🔴 高度敏感: 折扣品数量占比 >= 70%
- 🟡 中度敏感: 折扣品数量占比 >= 40% AND < 70%
- 🟢 低度敏感: 折扣品数量占比 < 40%
```

**折扣猎手** (Discount Seeker)

```
条件：折扣敏感度 = "高度敏感" AND 订单数 >= 3
```

#### 1.3 客户类型标签

**新老客** (New/Old Customer)

```
直接使用：client_monthly_tag
- 新客
- 老客
```

#### 1.4 活跃度标签（基于订单）

**购买频率**

```
计算逻辑：
- 首次购买时间 = MIN(最后付款时间)
- 最后购买时间 = MAX(最后付款时间)
- 购买天数跨度 = (最后购买时间 - 首次购买时间) / 天数
- 购买频率 = 总订单数 / 购买天数跨度 * 365（次/年）

分级：
- 高频: >= 5次/年
- 中频: 3-4次/年
- 低频: 1-2次/年
- 单次: 历史仅1次购买
```

**L6M活跃度** (近6个月)

```
计算：
- L6M订单数 = COUNT(最后付款时间 IN 最近6个月)
- L6M消费金额 = SUM(最后付款时间 IN 最近6个月的实际消费)
```

**历史LTV** (Lifetime Value)

```
计算：SUM(所有订单的实际消费金额)
```

#### 1.5 品类偏好标签

**品类偏好** (Category Preference)

```
计算逻辑：
- 按category分组统计每个品类的购买次数和金额
- 品类购买占比 = 品类订单数 / 总订单数
- Top品类 = 购买次数最多的品类

标签示例：
- "斗客" (Pipe类购买占比 >= 50%)
- "配件客" (配件类购买占比 >= 50%)
- "全能型" (多品类均衡)
```

#### 1.6 地理位置标签

**城市分布**

```
- 一线城市: 北京、上海、广州、深圳
- 新一线: 成都、杭州、重庆、武汉等
- 二线、三线、四线...
```

### 二、基于聊天记录的标签

#### 2.1 沟通活跃度

**聊天频率**

```
计算：
- 首次聊天时间 = MIN(msg_time WHERE user_nick = xxx)
- 最后聊天时间 = MAX(msg_time WHERE user_nick = xxx)
- 聊天数跨度 = (最后聊天 - 首次聊天) / 天数
- 总消息数 = COUNT(*)
- 聊天频率 = 总消息数 / 聊天数跨度（条/天）

分级：
- 非常活跃: >= 5条/天
- 活跃: 1-4条/天
- 一般: 0.1-0.9条/天
- 沉默: < 0.1条/天
```

**近期活跃度** (L30D - 最近30天)

```
计算：
- L30D消息数 = COUNT(msg_time IN 最近30天)
- 最后沟通时间 = MAX(msg_time)
```

**平均回复间隔**

```
计算：
- 买家的消息序列
- 计算相邻买家消息的时间差
- 平均间隔天数 = AVG(时间差) / 86400（转换为天）
```

#### 2.2 情绪倾向

**情绪分析** (Sentiment)

```
方法1: 关键词匹配（快速）
- 正向词: 谢谢、好、满意、喜欢、推荐...
- 负向词: 差、慢、贵、不好、失望、退货...
- SentimentScore = (正向词数 - 负向词数) / 总消息数

方法2: 智谱AI分析（准确）
- 将最近N条聊天记录发送给智谱AI
- 返回情绪评分（0-1）和分类（Positive/Neutral/Negative）

标签：
- 积极正面: Score >= 0.6
- 中性: 0.4 <= Score < 0.6
- 消极负面: Score < 0.4
```

#### 2.3 意图分析

**意图分布** (Intent Distribution)

```
使用关键词 + AI混合分析：

意图类型：
1. Pre-sale Inquiry (售前咨询)
   - 关键词: 适合、新手、怎么样、推荐、哪个好...

2. Post-sale Support (售后支持)
   - 关键词: 收到、用了、问题、坏了、怎么用...

3. Logistics (物流)
   - 关键词: 快递、发货、物流、到哪了、几天...

4. Usage Guide (使用指南)
   - 关键词: 怎么清洗、怎么用、如何保养...

5. Complaint (投诉)
   - 关键词: 投诉、差评、退货、退款、太慢...

计算方法：
- 方法1: 关键词匹配统计
- 方法2: 智谱AI批量分析

输出：雷达图数据
{
  "Pre-sale Inquiry": 85,
  "Post-sale Support": 45,
  "Logistics": 30,
  "Usage Guide": 60,
  "Complaint": 10
}
```

### 三、综合标签（订单 + 聊天）

#### 3.1 流失风险标签 ⭐ 重要

**流失买家** (Churn Risk)

```
定义（用户提供）：
条件1: 2年内无复购
  - 最后购买时间 < DATE_SUB(NOW(), INTERVAL 2 YEAR)

条件2: 近6个月无咨询
  - 最后聊天时间 < DATE_SUB(NOW(), INTERVAL 6 MONTH)

判定：
IF (条件1 AND 条件2) THEN 流失买家
```

**流失预警** (At-Risk)

```
条件（满足之一）：
- L6M无订单 AND L30D无咨询
- 订单频率明显下降（L6M订单数 < 历史平均的50%）
- 最近情绪负面 >= 60%
```

#### 3.2 客户生命周期标签

**潜力客户** (Potential)

```
条件：
- 新客 AND 近期咨询频繁（L30D消息 >= 10）
- 未下单 OR 下单金额 < 1,000
```

**成长客户** (Growing)

```
条件：
- 老客 AND L6M消费 > 历史LTV的20%
- L6M订单数 >= 2
```

**成熟客户** (Mature)

```
条件：
- LTV >= 10,000
- 复购次数 >= 3
- 近期有购买（L6M有订单）
```

#### 3.3 特殊标签

**大额客户** (Big Spender)

```
条件：
- 单笔订单 >= 10,000
- OR LTV >= 50,000
```

**礼品偏好** (Gift Buyer)

```
条件：
- 聊天中提到"送礼"、"礼物"、"朋友"等关键词 >= 3次
- OR 地址非本人（备注中有"xx收"）
```

**技术型买家** (Technical Enthusiast)

```
条件：
- 聊天中提到专业术语（材质、工艺、尺寸等）频率高
- 咨询商品细节参数 >= 5次
```

## 📈 数据计算流程

### Step 1: 数据抽取 (ETL)

```sql
-- 1. 从订单表聚合买家基础指标
SELECT
    买家昵称 as user_nick,
    COUNT(DISTINCT 订单号) as total_orders,
    SUM(成交总金额 - 退款金额) as historical_ltv,
    MIN(最后付款时间) as first_purchase_date,
    MAX(最后付款时间) as last_purchase_date,
    SUM(CASE WHEN FP_MD = 'MD' THEN 1 ELSE 0 END) as discount_order_count,
    SUM(CASE WHEN FP_MD = 'FP' THEN 1 ELSE 0 END) as fullprice_order_count,
    COUNT(DISTINCT 城市) as city_count,
    MAX(城市) as city,
    MAX(client_monthly_tag) as customer_type
FROM dunhill_t01_trade_line
GROUP BY 买家昵称;

-- 2. 从聊天表聚合沟通指标
SELECT
    user_nick,
    COUNT(*) as total_messages,
    MIN(msg_time) as first_chat_date,
    MAX(msg_time) as last_chat_date,
    COUNT(DISTINCT DATE(msg_time)) as chat_days
FROM chat_history
GROUP BY user_nick;

-- 3. 获取L6M指标
SELECT
    买家昵称 as user_nick,
    COUNT(DISTINCT 订单号) as l6m_orders,
    SUM(成交总金额 - 退款金额) as l6m_spend
FROM dunhill_t01_trade_line
WHERE 最后付款时间 >= DATE_SUB(NOW(), INTERVAL 6 MONTH)
GROUP BY 买家昵称;
```

### Step 2: 标签计算 (Python)

```python
# 标签计算逻辑
def calculate_buyer_tags(order_data, chat_data):
    tags = []

    # 1. VIP等级
    ltv = order_data['historical_ltv']
    orders = order_data['total_orders']
    if ltv >= 50000 or orders >= 20:
        tags.append("VIP5")
    elif ltv >= 20000 or orders >= 10:
        tags.append("VIP4")
    # ...

    # 2. 折扣敏感度
    discount_ratio = order_data['discount_order_count'] / orders
    if discount_ratio >= 0.7:
        tags.append("折扣猎手")
        tags.append("高度敏感")
    # ...

    # 3. 流失风险
    last_purchase = order_data['last_purchase_date']
    last_chat = chat_data['last_chat_date']
    if (days_since(last_purchase) > 730) and (days_since(last_chat) > 180):
        tags.append("流失风险")
    # ...

    return tags
```
