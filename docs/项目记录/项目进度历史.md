# SmokeSignal Analytics - 项目开发历程

**最后更新：** 2025-01-19
**当前版本：** v2.0 - 目标客户预计算优化版

---

## 📊 项目演进

### v1.0 - 基础版（已废弃）

**架构：**
- 直接查询 `dunhill_t01_trade_line` 视图
- 实时计算所有买家数据
- FastAPI后端 + React前端

**问题：**
- ❌ 查询超时（10-30秒）
- ❌ 数据范围过大（所有买家）
- ❌ 前端响应缓慢

**经验教训：**
- 复杂视图不适合实时查询
- 需要明确目标客户范围
- 预计算是性能关键

---

### v2.0 - 目标客户预计算版 ✅ **当前版本**

**架构升级：**
- ✅ 创建 `target_buyers_precomputed` 预计算表
- ✅ 聚焦高价值客户（SMOKER + VIC）
- ✅ 存储过程自动刷新（每天11:00）
- ✅ 新增 `/api/v2` 接口

**性能提升：**
- 🚀 查询速度提升 **40-120倍**
- 🚀 响应时间：10-30秒 → 0.1-0.3秒
- 🚀 识别391个目标买家（270 SMOKER + 103 VIC + 18 BOTH）

**关键改进：**
1. **客户范围优化**
   - SMOKER: 购买过Pipes/Lighters的高消费客户
   - VIC: Rolling 24M netsales >= 30K
   - 摒弃全量客户监控

2. **技术优化**
   - 32字段预计算表
   - 10个性能索引
   - 窗口函数优化client_monthly_tag
   - SQL文件分离（10个独立查询）

3. **数据准确性**
   - 修正client_monthly_tag获取逻辑
   - 使用最近购买记录的tag值
   - 100%数据准确性验证

## 🎉 总结

从v1.0到v2.0，我们学到了：
1. **性能优先** - 预计算比实时计算更可靠
2. **聚焦核心** - 目标客户比全量客户更有价值
3. **自动化** - 定时任务确保数据始终最新
4. **数据质量** - 字段逻辑的正确性至关重要

**当前状态：** ✅ 生产就绪，性能优秀，数据准确
