## 📊 项目演进

### v1.0 - 基础版（已废弃）

**架构：**
- 直接查询 `dunhill_t01_trade_line` 视图
- 实时计算所有买家数据
- FastAPI后端 + React前端

**问题：**
- ❌ 查询超时（10-30秒）
- ❌ 数据范围过大（所有买家）
- ❌ 前端响应缓慢

**经验教训：**
- 复杂视图不适合实时查询
- 需要明确目标客户范围
- 预计算是性能关键

---

### v2.0 - 目标客户预计算版 ✅ **当前版本**

**架构升级：**
- ✅ 创建 `target_buyers_precomputed` 预计算表
- ✅ 聚焦高价值客户（SMOKER + VIC）
- ✅ 存储过程自动刷新（每天11:00）
- ✅ 新增 `/api/v2` 接口

**性能提升：**
- 🚀 查询速度提升 **40-120倍**
- 🚀 响应时间：10-30秒 → 0.1-0.3秒
- 🚀 识别391个目标买家（270 SMOKER + 103 VIC + 18 BOTH）

**关键改进：**
1. **客户范围优化**
   - SMOKER: 购买过Pipes/Lighters的高消费客户
   - VIC: Rolling 24M netsales >= 30K
   - 摒弃全量客户监控

2. **技术优化**
   - 32字段预计算表
   - 10个性能索引
   - 窗口函数优化client_monthly_tag
   - SQL文件分离（10个独立查询）

3. **数据准确性**
   - 修正client_monthly_tag获取逻辑
   - 使用最近购买记录的tag值
   - 100%数据准确性验证

## 🤖 AI分析功能完善 (2025-01-20)

### 问题发现

用户反馈了三个关键问题：
1. **AI分析数据错误** - mylifemyrule客户历史消费6500元，但AI总结显示"历史消费总额为0"
2. **AI洞察质量不足** - AI仅重复已有数据（城市、消费金额），未提供深度洞察
3. **前端模块数据缺失** - Purchase History和Communication标签页显示无数据

### 根本原因分析

**问题1：字段名不匹配**
- API返回`historical_net_sales`，但AI提示词期望`historical_ltv`
- 影响：AI无法获取正确的消费数据
- 涉及文件：`backend/ai/zhipu_client.py:85`，`backend/api/target_routes.py:355`

**问题2：AI提示词未明确分析要求**
- 原提示词仅要求"总结客户信息"，未强调深度分析
- AI倾向重复已有数据，而非挖掘行为模式和心理动机
- 涉及文件：`backend/ai/zhipu_client.py:67-113`

**问题3：前端数据流断裂**
- 前端未调用orders和chats API获取详细数据
- `groupedMessages`使用错误数据源（`currentSession.messages`而非`enrichedProfile.chat_history`）
- 涉及文件：`src/App.tsx:610-650`（API调用），`src/App.tsx:699-710`（消息分组）

### 修复方案

**修复1：字段名回退机制**
```python
# 修改前
historical_ltv = float(profile.get('historical_ltv', 0))

# 修改后
historical_ltv = float(profile.get('historical_net_sales', profile.get('historical_ltv', 0)))
```

**修复2：AI提示词深度重写**
- 明确禁止："来自XX的客户，消费了XX元"
- 强调分析：购买决策模式、价值取向、消费心理特征
- 添加具体示例：好的summary vs 坏的summary
- 增强`order_summary`：包含L6M占比、平均客单价、退款率等深度指标

**修复3：前端数据流完整化**
```typescript
// 添加并行API调用
const [ordersRaw, chatsRaw] = await Promise.all([
  apiClient.getBuyerOrders(currentSession.user_nick, 50),
  apiClient.getBuyerChats(currentSession.user_nick, 100)
]);

// 数据转换：产品级→订单级
const orderMap = new Map<string, any>();
ordersRaw.forEach((item: any) => {
  // 按order_id分组聚合
});

// 修复groupedMessages数据源
const chatHistory = enrichedProfile?.chat_history || currentSession?.messages || [];
```

4. **数据质量** - 字段逻辑的正确性至关重要
