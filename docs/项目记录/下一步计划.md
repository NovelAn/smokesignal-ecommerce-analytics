## 🎯 短期改进计划（1-2周）

### 1. 前端已知问题修复 🔴 **高优先级**

**发现日期：** 2025-01-27

**1.1 Latest Purchase 显示逻辑**
- **位置：** Purchase History 视图
- **问题：** 当同一订单包含多个商品时，Latest Purchase 只显示其中一个商品
- **用户反馈：** "如果有多个商品在同一订单，显示不完整"
- **建议方案：**
  - 显示最近的订单，包含该订单的所有商品列表
  - 或显示最近的N个商品（不限制订单）
- **优先级：** 🔴 高

**1.2 商品图片加载错误处理**
- **位置：** 订单历史表格（Full Order History）
- **问题：** 部分图片链接失效或无法加载，显示破损图标
- **建议方案：**
  - 添加 onError 处理，显示占位图
  - 使用默认商品图标
- **优先级：** 🟡 中

**1.3 Dashboard 图表尺寸警告**
- **位置：** Overview 页面所有图表
- **问题：** 控制台警告 "The width(-1) and height(-1) of chart"
- **影响：** 图表可能显示不正常
- **建议方案：**
  - 为图表容器设置明确的宽度和高度
  - 使用 ResponsiveContainer 包裹图表
- **优先级：** 🟡 中

**1.4 数据加载状态指示器**
- **位置：** 全站
- **问题：** 缺少 Loading 状态，用户不知道数据是否正在加载
- **建议方案：**
  - 添加 Skeleton Screen 或 Spinner
  - 在数据加载时显示占位符
- **优先级：** 🔴 高

**1.5 API 错误提示**
- **位置：** API 调用处
- **问题：** API 失败时没有友好的错误提示
- **建议方案：**
  - 添加 Toast 或 Notification 组件
  - 显示具体的错误信息
- **优先级：** 🔴 高

**预期收益：** 用户体验提升30-40%
