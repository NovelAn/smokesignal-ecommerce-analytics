# client_monthly_tag 字段补充说明

## 📌 字段概述

**字段名**: `client_monthly_tag`
**数据类型**: `VARCHAR(50)`
**说明**: 新老客标识字段,直接来自源数据`dunhill_t01_trade_line.client_monthly_tag`
**取值**:
- `新客户` - 新客户
- `老客户` - 老客户

## 🎯 为什么这个字段很重要

1. **客户生命周期管理**: 区分新老客户是CRM的基础
2. **营销策略差异**:
   - 新客户: 需要教育、引导首次购买
   - 老客户: 需要维护、促进复购
3. **价值分析**: 新老客户的LTV、转化率等指标差异巨大
4. **服务优先级**: 老客户通常享有更高优先级

## 🔧 技术实现

### 数据来源

```sql
-- 直接从源表获取
MAX(t.client_monthly_tag) as client_monthly_tag
FROM dunhill_t01_trade_line t
```

**说明**:
- 使用`MAX()`聚合,因为一个买家可能有多条订单记录
- 取最新记录的`client_monthly_tag`值
- 如果源数据中该字段已正确维护,直接使用即可

### 索引优化

```sql
-- 为client_monthly_tag创建索引,加速筛选查询
CREATE INDEX idx_client_monthly_tag ON target_buyers_precomputed(client_monthly_tag);
```

### 查询性能

## 🔍 常见问题

### Q1: 为什么不自己计算新老客?

**A**: 源数据`client_monthly_tag`已经由业务系统维护,直接使用可以:
- 保持与业务系统一致
- 避免复杂的计算逻辑
- 减少数据不一致风险

### Q2: 新客户的定义标准是什么?

**A**: 这个标准由业务系统维护,可能基于:
- 首次购买时间
- 累计订单数
- 累计消费金额
- 或其他业务规则

具体标准请咨询业务团队或查看业务系统文档。

### Q3: 如果需要自定义新老客定义怎么办?

**A**: 可以在预计算表中添加计算字段:

```sql
-- 例如: 基于6个月内有订单定义为活跃客户
ALTER TABLE target_buyers_precomputed
ADD COLUMN is_active_last_6m BOOLEAN DEFAULT FALSE
COMMENT '近6个月活跃';

UPDATE target_buyers_precomputed
SET is_active_last_6m = CASE
    WHEN l6m_orders > 0 THEN TRUE
    ELSE FALSE
END;
```
