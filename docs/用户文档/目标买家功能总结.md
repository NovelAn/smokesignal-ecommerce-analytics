# 🎯 目标买家优化方案 - 实施总结

## 📌 问题诊断

### 原始问题
1. **查询超时**: `dunhill_t01_trade_line` VIEW查询复杂,每次请求JOIN多个大表
2. **前端响应慢**: 刷新Dashboard需要10-30秒,用户体验差
3. **数据范围过大**: 监控所有买家,而非聚焦高价值客户

### 性能瓶颈分析

```
dunhill_t01_trade_line VIEW结构:
├── merge_orders (大表)
├── dunhill_bi订单源 (大表)
├── dunhill_dtc订单源_hive (大表)
├── dunhill_tm退款源 (大表)
├── dunhill_dtc退款源_hive (大表)
├── p (产品表)
└── price (价格表)

每次查询JOIN 7+张大表 → 导致超时
```

---

## ✅ 解决方案

### 核心策略

**预计算 + 目标买家过滤 + SQL文件分离**

1. **目标买家识别**: 只监控Smoker(购买Pipes/Lighters)和VIC(Rolling 24M >= 30K)
2. **预计算表**: 每天上午11点自动更新所有指标
3. **SQL与代码分离**: 查询SQL独立存储,便于维护
4. **API版本管理**: 新增`/api/v2`路由,保持向后兼容

### 架构设计

```
┌─────────────────────────────────────────────────────────────┐
│                      前端 (React)                            │
└──────────────────────────┬──────────────────────────────────┘
                           │
                           ├─→ /api/v1/buyers (旧版,慢)
                           │   └─→ BuyerAnalyzer → VIEW查询 (10-30秒)
                           │
                           └─→ /api/v2/buyers (新版,快) ⭐
                               └─→ TargetBuyerAnalyzer
                                   └─→ TargetBuyerQueries
                                       └─→ SQL文件 → target_buyers_precomputed表
                                           (预计算, < 0.5秒)
```

## 🎯 核心特性

### 1. 智能买家过滤

**Smoker买家**:
- 精确品类匹配(`category IN ('Pipes', 'Lighters')`)
- 自动识别并标记为Smoker

**VIC买家**:
- Rolling 24个月净销售 >= 30,000
- 自动计算VIP等级(V3/V2/V1/V0)

**重叠客户**:
- 既是Smoker又是VIC的客户标记为`BOTH`
- 这部分是核心高价值客户

### 2. 丰富的预计算指标

#### 历史指标
- 历史GMV、退款、净销售
- 总订单数、有效订单数
- 退款率

#### 时间周期指标
- Rolling 24M (VIP计算专用)
- L6M (近6个月)
- L1Y (近1年)

#### 聊天指标
- 历史沟通频次(去重日期天数)
- 近3个月沟通频次
- 最后沟通时间

#### 智能标签
- VIP等级(V3/V2/V1/V0/Non-VIP)
- 折扣敏感度(高度/中度/低度)
- 流失风险(高/中/低)
- 品类偏好(TOP3)

## 🎓 关键技术点

### 1. 目标买家识别SQL

```sql
-- Smoker + VIC 识别(使用临时表和ON DUPLICATE KEY)
CREATE TEMPORARY TABLE tmp_target_buyers (
    buyer_nick VARCHAR(255) PRIMARY KEY,
    is_smoker BOOLEAN,
    is_vic BOOLEAN,
    buyer_type VARCHAR(50)
);

-- 插入Smoker买家
INSERT INTO tmp_target_buyers (buyer_nick, is_smoker, is_vic, buyer_type)
SELECT DISTINCT 买家昵称, TRUE, FALSE, 'SMOKER'
FROM dunhill_t01_trade_line
WHERE category IN ('Pipes', 'Lighters');

-- 插入VIC买家(重复则更新为BOTH)
INSERT INTO tmp_target_buyers (buyer_nick, is_smoker, is_vic, buyer_type)
SELECT DISTINCT 买家昵称, FALSE, TRUE, 'VIC'
FROM dunhill_t01_trade_line
WHERE 最后付款时间 >= DATE_SUB(NOW(), INTERVAL 24 MONTH)
GROUP BY 买家昵称
HAVING SUM(成交总金额 - IFNULL(退款金额, 0)) >= 30000
ON DUPLICATE KEY UPDATE
    is_vic = TRUE,
    buyer_type = 'BOTH';
```

### 2. 动态SQL构建

```python
# Python代码中的动态WHERE子句处理
conditions_to_remove = []

if not buyer_type:
    conditions_to_remove.append('AND buyer_type = %(buyer_type)s')

# 移除不需要的条件
for condition in conditions_to_remove:
    sql = sql.replace(f'[[{condition}]]', '')

# 清理[[ ]]标记
import re
sql = re.sub(r'\[\[|\]\]', '', sql)
```

### 3. 品类偏好计算(使用子订单号)

```sql
-- 正确做法:使用子订单号统计
WITH buyer_category_stats AS (
    SELECT
        买家昵称,
        category,
        COUNT(DISTINCT 子订单号) as category_orders,  -- ✅ 使用子订单号
        SUM(成交总金额 - IFNULL(退款金额, 0)) as category_spend
    FROM dunhill_t01_trade_line
    WHERE category IS NOT NULL AND category != ''
    GROUP BY 买家昵称, category
),
ranked_categories AS (
    SELECT
        买家昵称,
        category,
        ROW_NUMBER() OVER (PARTITION BY 买家昵称 ORDER BY category_orders DESC) as rank_num
    FROM buyer_category_stats
)
SELECT
    买家昵称,
    MAX(CASE WHEN rank_num = 1 THEN category END) as top_category,
    MAX(CASE WHEN rank_num = 2 THEN category END) as second_category,
    MAX(CASE WHEN rank_num = 3 THEN category END) as third_category
FROM ranked_categories
GROUP BY 买家昵称;
```
