# 定时刷新设置指南

## 问题说明

阿里云 RDS MySQL 默认禁用事件调度器（event_scheduler = OFF），且普通用户无权限修改。因此 MySQL 事件调度器无法自动执行。

## 解决方案

### 方案 1：在阿里云控制台启用事件调度器（推荐）

**优点**：无需额外配置，直接使用 MySQL 内置功能

**步骤**：
1. 登录阿里云控制台
2. 进入 **RDS 管理控制台** → 选择你的 RDS 实例
3. 点击左侧菜单 **参数设置**
4. 找到参数 **`event_scheduler`**
5. 将其值设置为 **`ON`**
6. 点击 **提交** 并确认
7. 等待参数生效（可能需要重启实例）

**验证**：
```sql
SHOW VARIABLES LIKE 'event_scheduler';
-- 应该显示: ON
```

### 方案 2：使用操作系统定时任务

#### Windows 任务计划程序

**创建定时任务**：
1. 打开 **任务计划程序**（Task Scheduler）
   - Win+R → 输入 `taskschd.msc` → 回车
2. 点击右侧 **创建任务**
3. **常规** 选项卡：
   - 名称: `SmokeSignal Target Buyers Refresh`
   - 选择 **不管用户是否登录都要运行**
   - 勾选 **使用最高权限运行**
4. **触发器** 选项卡：
   - 点击 **新建**
   - 选择 **每天**
   - 开始时间: `11:00:00`
5. **操作** 选项卡：
   - 点击 **新建**
   - 操作: **启动程序**
   - 程序/脚本: `D:\Work\ai\projects\smokesignal-ecommerce-analytics\scripts\refresh_cron.bat`
6. 点击 **确定** 保存

**手动测试**：
```cmd
cd D:\Work\ai\projects\smokesignal-ecommerce-analytics
scripts\refresh_cron.bat
```

#### Linux/Mac crontab

**创建定时任务**：
```bash
# 编辑 crontab
crontab -e

# 添加以下行（每天上午 11:00 执行）
0 11 * * * /path/to/smokesignal-ecommerce-analytics/scripts/refresh_cron.sh

# 保存并退出
```

**手动测试**：
```bash
chmod +x scripts/refresh_cron.sh
./scripts/refresh_cron.sh
```

### 方案 3：使用云函数或第三方定时服务

**选项**：
- **阿里云函数计算（FC）**: 创建定时触发器
- **GitHub Actions**: 使用 workflow 定时执行
- **Cron-job.org**: 免费的在线 cron 服务

**示例（GitHub Actions）**：
创建 `.github/workflows/refresh-data.yml`：
```yaml
name: Refresh Target Buyers Data

on:
  schedule:
    - cron: '0 3 * * *'  # UTC 3:00 = 北京时间 11:00
  workflow_dispatch:  # 允许手动触发

jobs:
  refresh:
    runs-on: ubuntu-latest
    steps:
      - name: Execute MySQL refresh
        run: |
          mysql -h ${{ secrets.DB_HOST }} -u ${{ secrets.DB_USER }} -p${{ secrets.DB_PASS }} ${{ secrets.DB_NAME }} -e "CALL refresh_target_buyers_precomputed();"
```

## 验证定时任务是否正常工作

### 检查最后更新时间
```sql
SELECT MAX(updated_at) as last_update
FROM target_buyers_precomputed;
```

### 查看事件执行历史（如果使用方案1）
```sql
SELECT * FROM information_schema.EVENTS
WHERE EVENT_NAME = 'event_refresh_target_buyers';
```

### 查看日志文件
- Windows: `refresh_log.txt`
- Linux/Mac: `/var/log/smokesignal_refresh.log`

## 推荐方案

**如果你有阿里云 RDS 管理权限**：使用方案 1（在控制台启用）

**如果没有管理权限**：使用方案 2（Windows 任务计划程序或 crontab）

**如果需要跨平台/远程执行**：使用方案 3（云函数或 GitHub Actions）

## 故障排查

### 1. Windows 任务计划程序未执行
- 检查任务历史记录
- 确认账号有执行权限
- 检查脚本路径是否正确
- 手动运行脚本测试

### 2. Linux crontab 未执行
- 检查 cron 服务: `systemctl status cron`
- 查看 cron 日志: `grep CRON /var/log/syslog`
- 确认脚本有执行权限: `ls -l scripts/refresh_cron.sh`
- 使用绝对路径

### 3. MySQL 事件未执行
- 确认 event_scheduler = ON
- 检查事件状态: SHOW EVENTS
- 确认时区设置正确
- 查看错误日志
